# Lumio Commercial Package Skeleton

Date: 2026-03-11
Audience: product lead, field team, internal commercial review
Purpose: define what the package is without locking unsupported pricing or maturity claims

## Package name

Lumio governed enterprise workspace preview

## What is being sold now

A buyer-facing enterprise workspace platform preview centered on:

- governed flow
- OA v1 nine-role model
- one workspace shell
- current-workspace depth
- role-aware control plane

## Ideal buyer profile

- enterprise operations owner
- workflow governance owner
- tenant or workspace admin stakeholder
- audit or compliance stakeholder
- sponsor looking for one governed workspace shell instead of fragmented tooling

## In-scope product elements

- Workspace Overview
- Organization & Workspace
- Members & Access
- Trial Join
- Request Center
- Approval Center
- Review Center
- Operations Console
- Integration & Readiness Center
- Policy & Governance Center
- Audit & Reporting Center

## In-scope technical posture

- web-first enterprise workspace platform
- OA v1 nine-role model
- Okta OIDC-only enterprise login target
- explicit fail-closed posture for unsupported or ambiguous enterprise writes

## Evaluation package shape

### Package A: Guided Preview Evaluation

Best for:

- buyer introduction
- architecture/governance review
- early admin/workflow evaluation

Includes:

- one-pager
- 15-minute and 30-minute demo talk tracks
- admin onboarding pack
- support / escalation pack
- proof pack

### Package B: Controlled Workspace Review

Best for:

- deeper buyer diligence
- internal admin/gov/audit review
- current-workspace path evaluation

Includes:

- everything in Package A
- stronger current-workspace walkthrough
- mutation-boundary explanation
- readiness and audit deep dive

## Support posture

Current package support should be described as:

- preview support posture
- field-guided evaluation support
- honest boundary explanation
- issue classification and escalation via support/escalation pack

Do not describe it as:

- full production support closure
- full rollout hypercare closure

## Deployment framing

Use this language:

- governed enterprise workspace preview
- current-workspace depth is growing
- `local_lab` is sandbox / preview
- not full production closure
- not full pilot closure

## What is intentionally deferred

- broad provider coverage beyond Okta OIDC
- price-sheet finalization
- reference-customer proof claims
- full production deployment closure claims
- full pilot closure claims
- unsupported live mutation breadth

## Internal qualification checklist

Before field use, confirm:

- the buyer understands preview maturity
- the team can explain `current workspace` vs `local_lab`
- the team can explain the OA v1 nine-role model
- the team can explain readiness, audit receipt, and evidence bundle honestly
- the team does not drift into unsupported provider or production claims
